# Contributing to Protocol: Void

Thank you for your interest in contributing to Protocol: Void!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/protocol-void.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes: `npm run dev`
6. Commit your changes: `git commit -m 'Add some feature'`
7. Push to the branch: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Code Style

- Use consistent indentation (spaces, not tabs)
- Follow existing code patterns
- Comment complex logic
- Keep functions focused and modular

## Game Design Contributions

- New rooms, characters, or storylines are welcome
- Maintain the cyberpunk noir aesthetic
- Ensure evidence chains are logical
- Test all new paths and endings

## Reporting Issues

When reporting bugs, please include:
- Steps to reproduce
- Expected behavior
- Actual behavior
- Browser and OS information

## Questions?

Open an issue for discussion or questions about contributing.
